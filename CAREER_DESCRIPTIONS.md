# Project Descriptions for Resume & LinkedIn

## Resume Description

**Smart Meal Planner & Nutrition Tracker** | *Flutter, Riverpod, Hive NoSQL*
- Developed an offline-first mobile application to help users track their daily caloric and macronutrient intake with a focus on UI/UX best practices and glassmorphism design.
- Engineered robust local data persistence using Hive, enabling the application to queue unsynced data and perform automatic background synchronization when internet connectivity is detected.
- Implemented state management using Riverpod to maintain highly optimized widget rebuilds and reactive real-time updates across multiple screens.
- Designed an interactive analytics dashboard using fl_chart to visualize a 7-day caloric trend and goal achievement statistics, providing users with AI-driven nutritional insights.

## LinkedIn Description

🚀 Excited to share my latest mobile application project: **Smart Meal Planner & Nutrition Tracker**! 

I built this offline-first application using Flutter to help students and professionals seamlessly track their daily diets, calorie goals, and macros without needing a constant internet connection. 

Key technical highlights:
🔹 **State Management**: Utilized `Riverpod` for highly scalable and compile-safe state handling.
🔹 **Offline-First Storage**: Implemented `Hive` (NoSQL) for blazing-fast local data persistence, allowing users to log meals anywhere and sync later!
🔹 **Data Visualization**: Integrated `fl_chart` to build an interactive dashboard showcasing 7-day caloric trends and macro completion rates.
🔹 **Premium UI/UX**: Crafted a modern interface featuring sleek glassmorphism cards, micro-animations with `flutter_animate`, and full dark/light mode support.

This project was a fantastic deep dive into offline synchronization strategies and scalable Flutter architecture. Check out the screenshots below! 👇
